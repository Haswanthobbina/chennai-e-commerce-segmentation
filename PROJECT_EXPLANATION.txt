Zepto & Blinkit Chennai Customer Segmentation Project - Complete Explanation

Overview
====================
This project analyzes Chennai customers of Zepto & Blinkit (quick commerce platforms) using K-Means clustering to identify 4 customer segments for personalized marketing strategies. 

Files:
├── zepto_blinkit_segmentation_2.ipynb  (Google Colab notebook - data gen, EDA, modeling)
├── app.py                            (Streamlit dashboard - interactive app)
├── chennai_customer_segments.csv     (2000 synthetic customer records)
├── kmeans_model.pkl / scaler.pkl / pca_model.pkl (ML artifacts)
├── cluster_metadata.json             (cluster names, features info)
├── requirements.txt                  (Python dependencies)
└── PNG visualizations (EDA outputs)

Colab Notebook: zepto_blinkit_segmentation_2.ipynb
==================================================
Purpose: Complete ML pipeline from scratch.

Steps:
1. Install/Import libraries (pandas, sklearn, plotly, etc.)

2. Generate synthetic dataset (2000 customers):
   - Realistic Chennai localities (Anna Nagar, T Nagar, etc.)
   - Behavioral features: monthly_orders, avg_order_value, reorder_rate, discount_usage_pct, etc.
   - Categorical: platform (Zepto/Blinkit/Both), membership, payment mode, etc.
   - Correlations injected: high-spenders have more orders/membership

3. EDA:
   - Distributions (histograms), correlation heatmap
   - Platform pie, locality bar, payment preferences
   - Outputs: eda_overview.png, correlation_heatmap.png

4. Feature Engineering:
   - 12 numerical features for clustering (monthly_orders to ratings_given)
   - StandardScaler normalization

5. Optimal K Selection:
   - Elbow method + Silhouette score (K=2 to 10)
   - Selected K=4 (best balance)
   - Outputs: optimal_k.png

6. Train K-Means (K=4):
   - Fit on scaled features
   - Silhouette ~0.083

7. PCA Visualization:
   - 2D projection of clusters
   - Centroids plotted
   - Output: cluster_pca.png

8. Cluster Profiling:
   - Mean values per cluster on key metrics
   - Human-readable names assigned:
     * 0: 💎 Premium Loyalists (high spend, frequent)
     * 1: 🧪 Occasional Browsers (low frequency)
     * 2: 🔥 Bargain Hunters (high discount usage)
     * 3: ⭐ Regular Mainstream (balanced)

9. Visualizations:
   - Radar charts: segment comparison
   - Platform preference stacked bar
   - Monthly spend boxplots
   - Outputs: radar_profiles.png, segment_analysis.png

10. Export artifacts for app.py

Streamlit App: app.py
======================
Purpose: Interactive dashboard using saved models/data.

Pages:
1. 📊 Dashboard:
   - KPIs (customers, avg orders/AOV/spend, membership rate)
   - Pie: segment distribution
   - Bar: avg spend by segment
   - Stacked bar: platform by segment
   - Boxplot: AOV distribution by segment
   - Heatmap: locality × segment matrix
   - Strategy cards for each segment

2. 🔮 Predict Segment:
   - Sliders/dropdowns for 12 features (matches model)
   - Live prediction + confidence score
   - Distance bar chart to all centroids
   - Recommended strategy

3. 📋 Data Explorer:
   - Filters: segment/platform/locality
   - Interactive scatter plots (any 2 numeric features)
   - Filtered data table
   - CSV download

Features:
- Dark GitHub theme CSS
- Cache @st.cache_data/@st.cache_resource for speed
- Model/data load validation (sidebar errors if missing)
- Debug panel for file paths

Key Insights (from profiling)
=============================
- Premium Loyalists: Target with VIP perks
- Occasional Browsers: Re-engagement campaigns
- Bargain Hunters: Discount-heavy promotions
- Regular Mainstream: Loyalty/subscription focus

Tech Stack:
===========
- Python 3.10+, pandas, scikit-learn, plotly, streamlit
- Synthetic data mimics real Q-Commerce patterns
- Production-ready: models saved, app deployable

To Run:
1. pip install -r requirements.txt
2. Ensure all .pkl/.csv/.json files present
3. streamlit run app.py
   → Opens http://localhost:8501

Scalability: Add real data, more features (RFM recency/frequency fully), hyperparameter tuning, silhouette-based validation.
